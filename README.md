# together

`together` is an orchestration skill pack for local AI agent fleets.

Core model:
- Codex = planner, coordinator, verifier, integrator
- other agents = workers
- routing = ready-first, capability-fit, fallback-aware

No benchmark theater. No hard-coded worship of one CLI. No automatic external execution.

## v0.2 goals

- agent-agnostic orchestration
- known-provider registry, not only PATH scan
- lightweight health checks
- cache + report generation
- department-style routing
- runtime failover memory

## Install

```bash
npx skills add https://github.com/kyoo-147/together_working
```

Install only main skill:

```bash
npx skills add https://github.com/kyoo-147/together_working --skill "together"
```

## Architecture

`together` splits local orchestration into four layers:

1. Registry
   Curated known-provider catalog. Capability hints only. Editable.
2. Discovery
   Checks whether commands exist on PATH and whether lightweight health checks pass.
3. Routing
   Chooses ready agents by task fit and department preference.
4. Reporting
   Writes local cache and markdown report for fast review and resume.

Codex keeps ownership of:
- plan
- task split
- execution approval gate
- verification
- final synthesis

## Health model

Provider states:
- `not-installed`
- `ready`
- `auth-required`
- `permission-denied`
- `installed-but-broken`
- `installed-unknown`

Checks stay cheap:
- command exists
- `--help` or `--version` works
- obvious auth/config errors
- obvious permission errors

No model calls. No benchmark. No token burn.

## Local outputs

Generated by doctor flow:

- `.together/cache/agent-registry.json`
- `.together/cache/last-known-good.json`
- `.together/cache/runtime-state.json`
- `.together/reports/agent-report.md`
- `.together/providers.override.json`

Cache and report files are local working artifacts and are ignored by git.
Override file is machine-local control for routing and provider behavior.

## Department model

- Planning: `codex`, `claude`
- Research: `gemini`, `agy`, `claude`
- Vision: `gemini`, `cmdc`, `kimi-code`
- Engineering: `codex`, `cmdc`, `amp`, `opencode`
- Review: `claude`, `codex`, `cmdc`
- Verification: `codex`, `claude`, `cmdc`
- Fallback: any ready agent with matching hints

## Scripts

Scan and print table:

```bash
python skills/together/scripts/discover-agents.py --format table
```

Write cache only:

```bash
python skills/together/scripts/write-registry.py
```

Write full cache + report:

```bash
python skills/together/scripts/doctor.py
```

Override providers or routing without code changes:

```bash
notepad .together/providers.override.json
```

Supported override knobs:
- disable provider
- rank adjust
- disable or add capability hints
- reorder task preference
- reorder department preference
- change cooldown seconds

- registry is curated, not exhaustive
- capability hints are not proof of superiority
- report is for routing hygiene, not benchmarking
- failover memory is lightweight cooldown state, not a scheduler
